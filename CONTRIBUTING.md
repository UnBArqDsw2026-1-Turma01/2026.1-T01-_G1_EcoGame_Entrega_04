# Guia de Contribuição

Ao contribuir para este repositório, por favor, discuta a mudança que deseja fazer via issue ou canais de comunicação do grupo antes de realizar a alteração.

## Relatando Bugs

Antes de criar um relatório de bug, verifique se o problema já não foi relatado. Ao criar a issue, inclua o máximo de detalhes possível:

1.  **Título claro e descritivo:** Para identificar o problema rapidamente.
2.  **Passos para reproduzir:** Descreva exatamente o que você fez para o erro aparecer.
3.  **Comportamento esperado vs. Comportamento observado:** O que deveria acontecer e o que realmente aconteceu.
4.  **Contexto:** Informe se o erro é constante ou intermitente e em qual ambiente/navegador ocorreu.

## Sugerindo Melhorias

Sugestões de melhorias e novas funcionalidades são rastreadas como **GitHub Issues**:

* Use um título descritivo.
* Forneça uma descrição passo a passo da melhoria sugerida.
* Explique por que essa melhoria seria útil para o projeto e para o jogador do EcoGame.
* Se possível, inclua screenshots ou rascunhos da ideia.

## Processo de Pull Request (PR)

Para garantir a qualidade do código e da documentação:

1.  **Atualize o README.md:** Se sua alteração mudar a interface, requisitos ou funcionamento, documente no README.
2.  **Versionamento:** Siga o esquema de versionamento definido pelo grupo (ex: SemVer).
3.  **Revisão:** Um Pull Request só deve ser mergeado após a revisão de, pelo menos, um ou dois outros membros do grupo.
4.  **Verificação:** Certifique-se de que todas as verificações automáticas (se houver) estão passando antes de solicitar o merge.

## Estilo e Conduta

* Siga o [Código de Conduta](CODE_OF_CONDUCT.md) em todas as suas interações.
* Mantenha o padrão de escrita e organização de arquivos estabelecido nas entregas anteriores.

---
*Este guia visa organizar o fluxo de trabalho acadêmico e garantir que todos os integrantes estejam alinhados com as mudanças no projeto.*
